2004年度上期には御社から絶大なご厚情を頂き、心から感謝しております。

  • 上期：かみき
  • 下期：しもき
  • 絶大：ぜつだい
  • 厚情：こうじょう

既存問題をうまく改善することで、御社からの発注拡大のチャレンジに立ち向かう覚悟でございます。

  • 発注：はっちゅう
  • 立ち向かう：ある事柄に対し，正面から解決に取り組む。

ご提案の＊＊プロジェクト開発における課題などの議論・交流はぜひよろしくお願いいたします。

  • 〜における：事物涉及的范围/地点，“在～”，“于～”，“在～方面”
  • 〜お願い申し上げます
  • 〜お願いいたします

ご連絡どうもありがとうございます。至急原因を調査し、結果が出る次第連絡致します。遅くとも明日は状況を報告します。

  • 報告：ほうこく
  • 至急：しきゅう

先日お願いした資料をまだ頂いておりません。明日まで送付できない場合、テストの進捗に影響を及ぼすことになりますので、至急対応よろしくお願いいたします。

タスクごとの備考欄には、各担当者の名前を記入し、返送してください。

  • 備考欄：びこうらん
  • 返送：へんそう

今回の開発委託範囲としては、システム設計から、プログラム設計、コーディング、単体テスト、結合テストまでですが、成果物としては、プログラム設計書と結合テスト成績書のみです。

  • 委託：いたく
  • 成果物：せいかぶつ

残業することで遅れを挽回します

  • 挽回：ばんかい

先週は主に仕様理解及びドキュメントの習得作業でした。

  • 習得：しゅうとく

新しいメンバーが追加されたので、製造方法とスケジュールの見直し及び作業の割り当てなどを行いました。

  • 割り当て：わりあて。assign。分派、分配。

今週は某機能の単体試験に着手する予定です。

  • 着手：ちゃくしゅ。动手，开始。

日限
完了日

  • 日限：にちげん 截止日

何かありましたら随時連絡してください。

  • 随時：ずいじ

変数を定数に変更する。

  • 変数：へんすう var
  • 定数：ていすう const

全角を半角に変更する

  • 全角：ぜんかく
  • 半角：はんかく

モジュールファイルの先頭に統一にヘッダー情報を記載する

  • 統一：とういつ
  • 記載：きさい

各SQL文にコメントをちゃんと入れること。

  • 文：ぶん

画面に表示される文字のフォント・サイズを統一する

山下さんも早いうちに瀋陽を訪問する予定。

  • 早いうちに

4月下旬山下さんと羽柴さんが瀋陽を訪問するように手配したい

  • 手配：てはい。筹备，安排。

メンテナンス作業の窓口を設けること。

  • 設ける：もうける

概要シートのB_SELECTメソッドが「DOメソッド仕様書」ではC_SELECTになっています

  • メソッド：method

帳票設計書シートにて金額のフィールドフォーマットの「ZZ,ZZZ,ZZZ,ZZ9」は11バイトですが「DOデータ定義仕様書」では12バイトになっています。

  • フィールド：field
  • バイト：bite 位&

帳票設計書シートにて、「XXXX年XXXX月度」は和暦ですか、西暦ですか。

  • 帳票：ちょうひょう
  • 和暦：われき
  • 西暦：せいれき

不正なデータが入力されたのが原因です。

  • 〜が原因です

文字化けが原因です。

  • 文字化け：もじばけ。乱码。

詳細は以下文中にて。

  • 文中：ぶんちゅう
  • にて：現代語の「で」に相当する語
    • 1、場所や時を表す。において。
    • 2、手段・方法・材料を表す。でもって。
    • 3、 原因・理由を表す。によって。

仕様書の〇〇ページには〜の記載がありますが、〜どの意味でしょうか。

概要には休止処理が記載されていますが、画面には対応項目がありません。追加する必要がありますか。

WebサーバーはNetscapeやApacheを使用し、DBとのデータの連携はSQL netを使用します。

  • 連携:れんけい

ホームディレクトリ

  • ディレクトリ：directory

テンポラリファイル用

  • テンポラリファイル: temporary file

バーションを1.0とする

  • 〜を〜とする：把…作为…

ワーク表へデータ格納

  • 格納：かくのう

ログイン画面に遷移する。

  • 遷移：せんい

パスワード更新処理。更新処理後成功か失敗のメッセージページを出力する。

ログイン認証処理、失敗の場合メッセージページを出力、認証できた場合はログイン者の権限によりメニュー + お知らせページを出力する。

  • 〜により（よって、よる）：表示事物产生的原因、方式，具有“由于”、“取决于”、“根据~” 等意思

Excel マクロ起動に失敗した場合にはエラー(ERR_No. 249)を表示し、シャシ番号にフォーカスを移し、画面のロック解除を行い処理を終了します。

  • シャシ：車台。車輛的車底主框架。
  • フォーカス：焦点

債務者への交付については、送達/ 送付/ 交付しないを選択可能とします。

アップロード用テーブルから正規部品テーブルにデータを登録する。登録済みデータはなにもしない。

REN_PAR.TS_MSTに登録する。それぞれのテーブルに登録したしコード数を表示する。「戻る」アンカーを1つもうけ、戻り先はファイル指定画面を指定する。

  • アンカー：⚓️。锚点。
  • もうけ：設け。

項目：ステータス

説明：未調査は部品登録したばかりのデータで、調査済は一回以上調査結果をアップロードした部品を意味する。

  • ステータス：status。

新パスワードと新パスワードの確認は同じであること。

  • 〜こと：命令

画面を非表示にする。

  • 〜にする：使～变为，使～成为

コンポの一覧は前画面の条件をみたした取引先一覧。

取引先をチェンジすることにより、その取引先のデータを表示する。

  • コンポ：复选框
  • 満たす：みたす

カレントの取引先

  • カレント：current

保存の時はボディーフレームの項目をヘッダーフレームの隠しフィールドに値をコピーしてからサブミットすれば情報が全て送信できる。

  • 隠しフィールド

エントリ業務ポタンを押下した場合は、帳票読み取りを実行し、対応するエントリ画面を表示します。

  • 押下：おうか

アンカー：成功した時は一覧画面に戻る。失敗した時は詳細画面に戻る。（ヒストリーバック）

  • ヒストリーバック：history back

項目長はD/Bに合わせる。

  • 〜に合わせる：使～一致

なお、限界試験などはデモシステムのため割愛する。

  • デモ：demo
  • 割愛：かつあい。惜しいと思うものを、思いきって手放したり省略したりすること。

負荷試験(ヒートラン) 。要求定義で連続実行は行わないと指定しているデモシステムであり、検査対象外とする。

  • 負荷：ふか。
  • ヒートラン：heat run。

インクリメント試験

  • インクリメント：increment。増量。

目視確認

  • もくしかくにん。目测。

試験項目：予約番号に番号が自動発番される（新規予約からリンクしてきた時のみ）

試験方法：新規予約からリンクしてきたときに表示された番号が、DBのテーブルに無いことを確認。

全ての画面において「プレビュー」ボタンを押すと、新しいテンプレートを利用した帳票が表示されることを確認します。

  • プレビュー：preview

試験項目：予約名称への入力

試験方法：文字数制限を越えて入力された時入力されないことを確認。

データベースには新規に追加するデータと重複するものが有る状態で、新規に追加する基本予約、施設、備品予約情報を入力し、予約基本情報入力画面において「登録・更新」ボタンを押す。

基本予約を除き、施設、備品のうち、重複するものについては、重複の旨を伝える情報を表示し、DBには変化を与えず、重複しないものについては、登録が完了し、DBにも追加されて、その旨結果が通知される。

  • 旨：むね。宗旨、目的
  • 与える：あたえる。
